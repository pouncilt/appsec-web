Our Services

Hybrid Analysis

Popular & well respected security report, Verizon Data Breach Investigations Report 2014, concluded that application vulnerabilities are the leading cause of security breaches. At 36%, application breaches are far ahead of cyber-espionage (22%), Point of Sale intrusions (14%), card skimming (9%), and every other category. Ponemon Institute's 2014 Cost of Data Breach study revealed that the average cost to a company was $3.5 million, 15% more than 2013. Will these costs continue to escalate? Are there preventive measures and controls?


Hybrid Analysis, a combination of source code analysis and dynamic analysis, commencing in the early stages of Software Development Life Cycle (SDLC) is one of the preventive measures and will help reduce overall costs.

Remediation Help

Application Security consultants not only help identify security vulnerabilities, but also help developers and application teams understand essential details about the identified vulnerabilities, how they can potentially impact KP and offer insight on how to remediate them.

Developers and application teams have direct access to Application Security experts. As questions arise, whether they relate to the installation and usage of a source analysis tool or how to resolve a Cross Site Scripting or a SQL injection flaw, developers can either reach out to their Application Security consultant or drop a line to kpis-application-security@kp.org.

Continuous Assessment for existing and emerging Threats

Security Threat Landscape is highly dynamic, so much so that there is no such thing as "perfectly secure". Facts speak for themselves, Mandiant 2014 Threat report reveals that 100% of victims have up-to-date antivirus software, 63% of the breaches are reported by third parties and advanced attackers are on the network for 243 median number of days before being detected.

It is imperative that at KP, among other things, we continuously assess and monitor our web applications in production for existings and emerging threats. Verizon Data Breach Investigations Report 2014, concluded that web application vulnerabilities are the leading cause of security breaches. At 36%, web application breaches are far ahead of cyber-espionage (22%), Point of Sale intrusions (14%), card skimming (9%), and every other category.

Application Attack Defense - Web Application Firewall

Web Aplications being the leading cause of security breaches (Source: 2014 Verizon DBIR), they receive a continuous barrage of attacks. If cyber criminals uncover a crack in our defenses, they can steal sensitive information. Web Application Firewall monitors web attacks and prevents costly data breaches and downtime. It also helps satisfy PCI 6.6 compliance.

At KP, Web Application Firewall is deployed in sniffing mode in all the production data centers and has the ability to monitor all internet and intranet web traffic. Attacks on internet facing applications can potentially be blocked.
